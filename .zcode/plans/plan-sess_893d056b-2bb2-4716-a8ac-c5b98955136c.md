# 贴图就地标注编辑功能

## 目标
贴图面板（PinPanel）增加「编辑」按钮，点击后进入就地标注模式：工具栏浮在顶部，直接在贴图上画矩形/箭头/椭圆/文字/画笔/荧光笔/马赛克/橡皮，完成后烘焙写回贴图。

## 坐标正确性（已验证，复用现有标注系统）
`AnnotationCanvasView` 把标注存为**原图坐标**（`imagePoint(from:)` 转换），绘制时按 `scaleX/scaleY` 还原；`renderFlattenedImage` 按原图原生像素烘焙。因此标注与贴图显示尺寸无关，缩放后仍正确。无需自己处理坐标映射。

## 改动（PinPanelController.swift，单文件 + Localization.swift）

### 1. 新增编辑按钮
- 在 `closeButton` 旁加 `editButton`（SF Symbol `pencil.circle.fill`），约束紧邻关闭按钮左侧。
- 点击进入编辑模式（`enterEditMode()`）。

### 2. 编辑模式状态机（PinPanel）
新增状态 `isEditing: Bool`、`annotationModel: AnnotationCanvasModel`、`canvasView: AnnotationCanvasView?`、`editToolbar: NSView?`。

**进入编辑（`enterEditMode`）**：
- 保存当前 `rotationDegrees`，临时置 0（旋转下标注坐标会错）。
- 暂存当前 `pinView`，创建 `AnnotationCanvasView(baseImage: 当前image, model: annotationModel, contentMode: .fill)`，填满容器，替换 pinView。
- 顶部添加轻量 AppKit 工具栏（`PinEditToolbar`）：工具按钮组（selection/rectangle/arrow/ellipse/text/pencil/highlighter/mosaic/eraser）、颜色选取（几个常用色）、线宽、撤销、重做、清除、完成。
- 进入编辑时：`isMovableByWindowBackground = false`（防止拖动干扰标注），隐藏缩放（`scrollWheel` 在编辑模式下不处理缩放，只转发给 canvas）。
- 旋转的 R 键、缩放在编辑模式下禁用。

**退出编辑（`exitEditMode(save:)`）**：
- 若 save：`AnnotationCanvasView.flatten(baseImage: 当前image, model: annotationModel)` → 得到新 image → 用 `replaceBaseImage` 思路更新（更新 pinView.image，保存为新的 baseImage 供下次编辑）。
- 移除 canvasView + 工具栏，恢复 pinView 显示。
- 恢复 `rotationDegrees`（编辑前保存的值）、`isMovableByWindowBackground = true`。
- `annotationModel.resetSession()`。

### 3. baseImage 管理
- 新增 `private var currentImage: NSImage`（初始 = init 的 image，编辑后更新）。
- `pinView.image` 与 `currentImage` 同步。
- 每次编辑基于 `currentImage`（叠加之前标注的结果），支持反复编辑。

### 4. 工具栏（新写轻量 AppKit `PinEditToolbar`）
- 纯 AppKit `NSView`，横向排列按钮。避免引入 SwiftUI + ViewModel 耦合。
- 工具按钮点击 → `annotationModel.selectedTool = tool` + `lineWidth = tool.defaultLineWidth`。
- 颜色 → `annotationModel.strokeColor`。
- 线宽加减 → `annotationModel.lineWidth`。
- 撤销/重做 → `model.undo()/redo()` + `canvasView.needsDisplay = true`。
- 清除 → `model.clearAnnotations()`。
- 完成 → 回调 `onFinish`。
- 选中态高亮当前工具。

### 5. 本地化（Localization.swift）
新增 key：`screenshotEdit`（编辑）、`screenshotEditDone`（完成）、`screenshotClearAnnotations`（清除标注）。复用已有的 `screenshotTool*`、`screenshotUndo`、`screenshotRedo`、`screenshotLineWidth`。

## 不做的事
- 不弹独立窗口（用户明确选就地标注）。
- 不改贴图的拖动、缩放、透明度、旋转的现有逻辑（仅在编辑模式下临时禁用）。
- 不新增 OCR/复制到贴图内（纯标注）。
- 编辑期间禁用旋转（避免坐标错乱），退出后恢复。

## 边界处理
- 编辑模式下手势冲突：禁用 `isMovableByWindowBackground`、缩放；canvas 接管鼠标。
- 文字工具：`AnnotationCanvasView` 内联文本在非 `CaptureAnnotationPanel` 窗口下也能工作（agent 已确认 soft cast 跳过，firstResponder 仍转移）。
- 编辑期间窗口尺寸固定（不缩放），标注存原图坐标，所以即使之后缩放也正确。

## 验证
- `swiftc -typecheck` 全模块 0 错误。
- 真机：贴图→编辑→画标注→完成→标注正确出现在贴图上；再次编辑可叠加；编辑期间不可拖动/缩放/旋转。
